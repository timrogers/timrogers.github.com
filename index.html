Lloyds TSB screen scraper

I bank with Lloyds TSB - I have my current account and credit card with them. Like most online banking services though, they're not to up-to-date on APIs and the like. After looking around online, I found that there were a couple of scripts that people had built, but I didn't have much luck with them myself. So I decided to build my own screen scraper.

Once again using Ruby with Mechanize, much like my UCAS one (below), I've built a script which can login to Lloyds TSB with your username, password and memorable word. There were some particularly interesting challenges, especially with the memorable word part, since the login pages ask for particular characters which change each time.

When run, the script returns a list of your accounts and the recent transactions for them. If you want to give it a go, you can download the script here.

UCAS screen scraper

In the UK, we make university applications via a system called UCAS. It has no APIs and generally the whole experience with staying up to date with your application isn't too great. So I decided to build a solution comprised of two parts:

  • ucas, a Ruby script to be run periodically (every minute, for me!) which screen scrapes from the UCAS Track system using Mechanize and saves to a Redis database, providing SMS and email notifications.
  • ucas-frontend, a Sinatra app which provides a pretty interface to the Redis backend meaning that anyone can see my application status. It's running live at http://ucas.timrog.net.

All of the code is reusable for the project, and I'd love to see people contribute. For a bit more information on how things work, check out my blog post UCAS, universities and open data.

Unistats API library (for Ruby)

The UK government organisation Higher Education Funding Council for England provides a site called Unistats with lots of information about university courses in the UK. This even has a free-to-use API.

The organisation of the data is a bit odd, but I nonetheless decided to build an API library for my own use - you can find the details here, and install it really quickly by running gem install unistats.

Impatient (a Ruby gem)

I run quite a few projects on Heroku, and I get tired of the wait after deploy for them to be ready to go. It's really short, but I get annoyed every time I have to run heroku ps after deploying whilst I wait to check that everything has worked.

With my Ruby gem "impatient", this is no longer necessary. Basically, it lets you deploy your Heroku project simply by running imp - this runs the git push heroku master command and then repeatedly executes heroku ps until the app is up and ready to go. It'll then automatically open the site for you. Neat, huh?

You can download the source of this from my GitHub repository, or simply run gem install impatient.

Windows Phone application samples

As part of a number of competitions that Microsoft UK ran early in 2012 with prizes like Xbox 360s and Windows Phones, I built some really simple (but pretty cool!) Windows Phone apps and open sourced them. People are welcome to do as they please with these, and they serve as a great starting point for your own application, whether there's a competition to be won or not.

There are two sample apps available:

  • joker-soundboard-wp7, a soundboard featuring sounds from The Joker in awesome film The Dark Knight. You can adapt this to make any soundboard you wish, and it even includes the ability to set the sounds as a ringtone. This has been fairly commercially successful, as it happens.
  • strong>business-card-wp7, is a really simple application which basically acts a business card for your phone, including an oh-so-pointless QR code (but everyone loves them...). It's basically a picture and contact details. It's a bit pointless, but it still serves as an interesting demo.
